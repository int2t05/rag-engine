# 文档上传与向量化流程说明

## 概述

文档从上传到可检索经历以下阶段：

```
上传(MinIO临时) → 预览(可选) → 后台处理 → 向量化(Chroma/Qdrant) → 可检索
```

系统支持 **PDF、DOCX、MD、TXT** 四种格式，通过 LangChain 的文档加载器和分块器处理。

## 阶段详解

### 阶段1：上传到 MinIO 临时存储

**API**: `POST /api/knowledge-base/{kb_id}/documents/upload`

```python
# backend/app/api/api_v1/knowledge_base.py
@router.post("/{kb_id}/documents/upload")
async def upload_kb_documents(kb_id: int, files: List[UploadFile], ...):
    kb = db.query(KnowledgeBase).filter(
        KnowledgeBase.id == kb_id,
        KnowledgeBase.user_id == current_user.id
    ).first()
    if not kb:
        raise HTTPException(status_code=404, detail="未找到知识库")

    results = []
    for file in files:
        # 1. 计算文件 SHA-256 哈希
        file_content = await file.read()
        file_hash = hashlib.sha256(file_content).hexdigest()

        # 2. 检查完全相同文件是否已存在（文件名+hash相同则跳过）
        existing_document = db.query(Document).filter(
            Document.file_name == file.filename,
            Document.file_hash == file_hash,
            Document.knowledge_base_id == kb_id,
        ).first()
        if existing_document:
            results.append({
                "document_id": existing_document.id,
                "status": "exists",
                "skip_processing": True,
            })
            continue

        # 3. 上传到 MinIO 临时目录
        temp_path = f"kb_{kb_id}/temp/{file.filename}"
        minio_client.put_object(
            bucket_name=settings.MINIO_BUCKET_NAME,
            object_name=temp_path,
            data=file.file,
            length=len(file_content),
            content_type=file.content_type,
        )

        # 4. 创建 DocumentUpload 记录
        upload = DocumentUpload(
            knowledge_base_id=kb_id,
            file_name=file.filename,
            file_hash=file_hash,
            file_size=len(file_content),
            content_type=file.content_type,
            temp_path=temp_path,
        )
        db.add(upload)
        db.commit()

        results.append({
            "upload_id": upload.id,
            "file_name": file.filename,
            "temp_path": temp_path,
            "status": "pending",
            "skip_processing": False,
        })

    return results
```

**返回示例**:
```json
[
  {
    "upload_id": 1,
    "file_name": "report.pdf",
    "temp_path": "kb_1/temp/report.pdf",
    "status": "pending",
    "skip_processing": false
  }
]
```

### 阶段2：预览分块（可选）

**API**: `POST /api/knowledge-base/{kb_id}/documents/preview`

用户可在正式处理前预览分块效果，调整 `chunk_size` 和 `chunk_overlap`。

```python
# backend/app/services/document_processor.py
async def preview_document(
    file_path: str, chunk_size: int = 1000, chunk_overlap: int = 200
) -> PreviewResult:
    minio_client = get_minio_client()
    _, ext = os.path.splitext(file_path)
    ext = ext.lower()

    # 下载到临时文件（Windows 兼容处理）
    temp_file = tempfile.NamedTemporaryFile(delete=False, suffix=ext)
    temp_path = temp_file.name
    temp_file.close()

    minio_client.fget_object(
        bucket_name=settings.MINIO_BUCKET_NAME,
        object_name=file_path,
        file_path=temp_path,
    )

    try:
        # 根据扩展名选择加载器
        if ext == ".pdf":
            loader = PyPDFLoader(temp_path)
        elif ext == ".docx":
            loader = Docx2txtLoader(temp_path)
        elif ext == ".md":
            loader = UnstructuredMarkdownLoader(temp_path)
        else:
            loader = TextLoader(temp_path, encoding="utf-8")

        # 递归字符分块器
        documents = loader.load()
        text_splitter = RecursiveCharacterTextSplitter(
            chunk_size=chunk_size,
            chunk_overlap=chunk_overlap,
            length_function=len,
            separators=["\n\n", "\n", " ", ""],
        )
        chunks = text_splitter.split_documents(documents)

        preview_chunks = [
            TextChunk(content=chunk.page_content, metadata=chunk.metadata)
            for chunk in chunks
        ]
        return PreviewResult(chunks=preview_chunks, total_chunks=len(chunks))
    finally:
        os.unlink(temp_path)  # 清理临时文件
```

**分块策略**:
- `RecursiveCharacterTextSplitter`：递归地按 `\n\n` → `\n` → ` ` → `""` 的顺序切分，保留段落完整性
- `chunk_size=1000`：每个块约 1000 个字符（可配置）
- `chunk_overlap=200`：块之间有 200 字符重叠，避免边界截断丢失上下文

### 阶段3：触发后台处理

**API**: `POST /api/knowledge-base/{kb_id}/documents/process`

```python
@router.post("/{kb_id}/documents/process")
async def process_kb_documents(
    kb_id: int,
    upload_results: List[dict],
    background_tasks: BackgroundTasks,
    db: Session = Depends(get_db),
    current_user: User = Depends(get_current_user),
):
    kb = db.query(KnowledgeBase).filter(
        KnowledgeBase.id == kb_id,
        KnowledgeBase.user_id == current_user.id
    ).first()
    if not kb:
        raise HTTPException(status_code=404, detail="未找到知识库")

    # 跳过已存在的文件
    upload_ids = [r["upload_id"] for r in upload_results if not r.get("skip_processing")]
    if not upload_ids:
        return {"tasks": []}

    # 创建 ProcessingTask 记录
    uploads = db.query(DocumentUpload).filter(DocumentUpload.id.in_(upload_ids)).all()
    all_tasks = []
    for upload in uploads:
        task = ProcessingTask(
            document_upload_id=upload.id,
            knowledge_base_id=kb_id,
            status="pending"
        )
        all_tasks.append(task)

    db.add_all(all_tasks)
    db.commit()
    for task in all_tasks:
        db.refresh(task)

    # 添加后台任务（不阻塞响应）
    task_data = [
        {
            "task_id": task.id,
            "upload_id": upload.id,
            "temp_path": upload.temp_path,
            "file_name": upload.file_name,
        }
        for task, upload in zip(all_tasks, uploads)
    ]
    background_tasks.add_task(add_processing_tasks_to_queue, task_data, kb_id)

    return {"tasks": [{"upload_id": t["upload_id"], "task_id": t["task_id"]} for t in task_data]}
```

### 阶段4：后台向量化（核心）

**函数**: `process_document_background()` in `document_processor.py`

```python
async def process_document_background(
    temp_path: str, file_name: str, kb_id: int, task_id: int,
    db: Session = None,
    chunk_size: int = 1000, chunk_overlap: int = 200,
) -> None:
    # 创建数据库会话
    if db is None:
        db = SessionLocal()
        should_close_db = True

    task = db.query(ProcessingTask).get(task_id)
    task.status = "processing"
    db.commit()

    try:
        # 1. 从 MinIO 下载到本地临时目录
        local_temp_path = f"/tmp/temp_{task_id}_{file_name}"  # ⚠️ Windows 问题
        minio_client.fget_object(
            bucket_name=settings.MINIO_BUCKET_NAME,
            object_name=temp_path,
            file_path=local_temp_path,
        )

        # 2. 加载和分块文档
        _, ext = os.path.splitext(file_name)
        if ext == ".pdf":
            loader = PyPDFLoader(local_temp_path)
        elif ext == ".docx":
            loader = Docx2txtLoader(local_temp_path)
        elif ext == ".md":
            loader = UnstructuredMarkdownLoader(temp_path)  # ⚠️ BUG: 应该是 local_temp_path
        else:
            loader = TextLoader(local_temp_path, encoding="utf-8")

        documents = loader.load()
        text_splitter = RecursiveCharacterTextSplitter(
            chunk_size=chunk_size, chunk_overlap=chunk_overlap
        )
        chunks = text_splitter.split_documents(documents)

        # 3. 初始化向量存储（Chroma 或 Qdrant）
        embeddings = EmbeddingsFactory.create()
        vector_store = VectorStoreFactory.create(
            store_type=settings.VECTOR_STORE_TYPE,      # 配置: chroma 或 qdrant
            collection_name=f"kb_{kb_id}",              # 每个知识库一个 collection
            embedding_function=embeddings,              # OpenAI/DashScope/Ollama/ZhipuAI
        )

        # 4. 移动文件到 MinIO 永久目录
        permanent_path = f"kb_{kb_id}/{file_name}"
        source = CopySource(settings.MINIO_BUCKET_NAME, temp_path)
        minio_client.copy_object(
            bucket_name=settings.MINIO_BUCKET_NAME,
            object_name=permanent_path,
            source=source,
        )
        minio_client.remove_object(settings.MINIO_BUCKET_NAME, object_name=temp_path)

        # 5. 创建 Document 记录
        document = Document(
            file_name=file_name,
            file_path=permanent_path,
            file_hash=task.document_upload.file_hash,
            file_size=task.document_upload.file_size,
            content_type=task.document_upload.content_type,
            knowledge_base_id=kb_id,
        )
        db.add(document)
        db.commit()

        # 6. 创建 DocumentChunk 记录并存储到向量数据库
        for i, chunk in enumerate(chunks):
            chunk_id = hashlib.sha256(
                f"{kb_id}:{file_name}:{chunk.page_content}".encode()
            ).hexdigest()

            chunk.metadata.update({
                "source": file_name,
                "kb_id": kb_id,
                "document_id": document.id,
                "chunk_id": chunk_id,
            })

            doc_chunk = DocumentChunk(
                id=chunk_id,
                document_id=document.id,
                kb_id=kb_id,
                file_name=file_name,
                chunk_metadata={"page_content": chunk.page_content, **chunk.metadata},
                hash=hashlib.sha256(
                    (chunk.page_content + str(chunk.metadata)).encode()
                ).hexdigest(),
            )
            db.add(doc_chunk)

            if i > 0 and i % 100 == 0:
                db.commit()  # 每100条提交一次

        # 7. 添加到向量存储
        vector_store.add_documents(chunks)

        # 8. 更新状态
        task.status = "completed"
        task.document_id = document.id
        task.document_upload.status = "completed"
        db.commit()

    except Exception as e:
        task.status = "failed"
        task.error_message = str(e)
        db.commit()
    finally:
        os.remove(local_temp_path)  # 清理本地临时文件
        if should_close_db:
            db.close()
```

## 增量更新机制

```python
# backend/app/services/document_processor.py - process_document()
async def process_document(file_path, file_name, kb_id, document_id, ...):
    # 通过 SHA-256 哈希比对实现增量更新
    existing_hashes = chunk_manager.list_chunks(file_name)  # 当前DB中该文件的哈希集合

    new_chunks = []
    current_hashes = set()

    for chunk in preview_result.chunks:
        chunk_hash = hashlib.sha256((chunk.content + str(chunk.metadata)).encode()).hexdigest()
        current_hashes.add(chunk_hash)

        if chunk_hash not in existing_hashes:  # 新块
            new_chunks.append({...})
        # else: 块未变化，跳过

    # 新块入库
    if new_chunks:
        chunk_manager.add_chunks(new_chunks)
        vector_store.add_documents(documents_to_update)

    # 删除已移除的块
    chunks_to_delete = chunk_manager.get_deleted_chunks(current_hashes, file_name)
    if chunks_to_delete:
        chunk_manager.delete_chunks(chunks_to_delete)
        vector_store.delete(chunks_to_delete)
```

## 数据库模型

```python
# backend/app/models/knowledge.py

class KnowledgeBase(Base, TimestampMixin):
    __tablename__ = "knowledge_bases"
    id = Column(Integer, primary_key=True)
    name = Column(String(255), nullable=False)
    description = Column(LONGTEXT)
    user_id = Column(Integer, ForeignKey("users.id"), nullable=False)
    documents = relationship("Document", back_populates="knowledge_base", cascade="all, delete-orphan")

class Document(Base, TimestampMixin):
    __tablename__ = "documents"
    id = Column(Integer, primary_key=True)
    file_path = Column(String(255))         # MinIO 永久路径
    file_name = Column(String(255))          # 原始文件名
    file_hash = Column(String(64), index=True)  # SHA-256 去重
    file_size = Column(BigInteger)
    content_type = Column(String(100))
    knowledge_base_id = Column(Integer, ForeignKey("knowledge_bases.id"))
    chunks = relationship("DocumentChunk", back_populates="document", cascade="all, delete-orphan")

class DocumentChunk(Base, TimestampMixin):
    __tablename__ = "document_chunks"
    id = Column(String(64), primary_key=True)   # SHA-256(content)
    kb_id = Column(Integer, ForeignKey("knowledge_bases.id"))
    document_id = Column(Integer, ForeignKey("documents.id"))
    file_name = Column(String(255))
    chunk_metadata = Column(JSON)               # 包含 page_content 和元数据
    hash = Column(String(64), index=True)       # 增量更新用

class ProcessingTask(Base):
    __tablename__ = "processing_tasks"
    id = Column(Integer, primary_key=True)
    knowledge_base_id = Column(Integer, ForeignKey("knowledge_bases.id"))
    document_id = Column(Integer, ForeignKey("documents.id"), nullable=True)
    document_upload_id = Column(Integer, ForeignKey("document_uploads.id"), nullable=True)
    status = Column(String(50), default="pending")  # pending -> processing -> completed/failed
    error_message = Column(Text)
```

## 业务改进建议

### 🔴 高优先级

| 问题 | 风险 | 建议 |
|------|------|------|
| **Windows 临时路径硬编码为 `/tmp/`** (`process_document_background` 第298行) | Windows 系统上 `/tmp/` 通常不存在，下载失败 | 改用 `tempfile.gettempdir()` 或 `os.environ.get("TMP", tempfile.gettempdir())` |
| **后台任务无重试机制** | 网络波动导致失败后需手动重处理 | 引入 Redis/RabbitMQ 队列，设置最大重试次数（如3次），失败后通知用户 |
| **只加载第一个 KB 的向量存储用于检索** (chat_service.py:99) | 多知识库对话时其他 KB 被忽略 | 实现多库检索合并（RRF 融合或拼接得分） |

### 🟡 中优先级

| 问题 | 风险 | 建议 |
|------|------|------|
| `process_document_background` 中 `UnstructuredMarkdownLoader` 传错路径变量 | Bug:第325行用的是 `temp_path` 而非 `local_temp_path` | 修复路径变量统一使用 `local_temp_path` |
| `ChunkRecord` 每次实例化都创建新的数据库 engine | 连接资源浪费，多实例竞争 | 改为单例模式或传入 db session |
| 无处理进度实时推送 | 用户需轮询 `tasks` 接口 | 改用 WebSocket 或 SSE 实时推送进度 |
| 未处理文件大小限制 | 大文件可能导致 OOM 或超时 | 添加单文件大小上限（如 50MB）|

### 🟢 低优先级

| 问题 | 建议 |
|------|------|
| 支持更多文件格式 | 添加 PPT、Excel、HTML 等格式支持 |
| 分块参数默认值固定 | 提供知识库级别的分块配置 |
| 无文档删除清理 | 删除文档后立即清理向量数据（当前只清理 KB 时才清理） |
