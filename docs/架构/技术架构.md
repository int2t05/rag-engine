```
┌─────────────────────┐
│   Routes (HTTP)     │  ← 接收请求，参数校验
├─────────────────────┤
│   Service           │  ← 业务逻辑编排
├─────────────────────┤
```

## 关键技术设计

### 1. ContextVar 注入

用户 LLM/Embedding 配置通过 `ContextVar` 在异步请求/后台任务中传递：

```python
# app/shared/ai_runtime_context.py
ai_runtime_settings_var: ContextVar[AiRuntimeSettings | None] = ContextVar(
    "ai_runtime_settings", default=None
)

# 使用方式
set_ai_runtime_token(settings)  # 设置
reset_ai_runtime_token()         # 清理
```

### 2. ai_runtime_scope

`contextmanager` 封装，自动设置/清理 ContextVar：

```python
# app/shared/ai_runtime_scope.py
@contextmanager
def ai_runtime_scope(settings: AiRuntimeSettings):
    token = set_ai_runtime_token(settings)
    try:
        yield
    finally:
        reset_ai_runtime_token(token)
```

### 3. LCEL 管道

LangChain Expression Language 组合 RAG 链：

```python
chain = prompt | llm | StrOutputParser()
```

### 4. SSE 流式响应

```javascript
// 格式
data: {"text": "<base64_ctx>__LLM_RESPONSE__<文本块>"}
```

## 外部服务依赖

| 服务 | 用途 | 配置 |
|------|------|------|
| MySQL | 关系型数据 | `MYSQL_*` 环境变量 |
| MinIO | 文档对象存储 | `MINIO_*` 环境变量 |
| ChromaDB | 向量数据库 | `CHROMA_DB_HOST/PORT` |
| OpenAI/Ollama | LLM + Embeddings | 用户在 DB 中配置 |
