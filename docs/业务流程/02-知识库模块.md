## 模块概述

知识库模块负责文档的上传、解析、分块、向量化存储和检索测试。

**源码位置**: `app/modules/knowledge/`

## API 端点

### 知识库 CRUD

| 方法 | 端点 | 功能 |
|------|------|------|
| POST | `/api/knowledge-base` | 创建知识库 |
| GET | `/api/knowledge-base` | 知识库列表 |
| GET | `/api/knowledge-base/{kb_id}` | 知识库详情 |
| PUT | `/api/knowledge-base/{kb_id}` | 更新元数据 |
| DELETE | `/api/knowledge-base/{kb_id}` | 删除知识库 |

### 文档与检索

| 方法 | 端点 | 功能 |
|------|------|------|
| POST | `/api/knowledge-base/{kb_id}/documents/upload` | 多文件上传 |
| POST | `/api/knowledge-base/{kb_id}/documents/preview` | 预览分块 |
| POST | `/api/knowledge-base/{kb_id}/documents/process` | 提交解析与向量化任务 |
| POST | `/api/knowledge-base/cleanup` | 清理临时上传（无时间门槛；规则见 [架构/API路由.md](../架构/API路由.md)） |
| GET | `/api/knowledge-base/{kb_id}/documents/tasks` | 查询处理任务状态（`task_ids` 查询参数） |
| POST | `/api/knowledge-base/{kb_id}/documents/{doc_id}/replace` | 同名重新上传（`multipart` 字段 `file`），Query 可选 `chunk_size`、`chunk_overlap`（默认 1000 / 200，须重叠小于块大小）；覆盖 MinIO 原路径并调用 `process_document` 增量更新向量 |
| GET | `/api/knowledge-base/{kb_id}/documents/{doc_id}` | 文档详情 |
| DELETE | `/api/knowledge-base/{kb_id}/documents/{doc_id}` | 删除文档 |
| POST | `/api/knowledge-base/{kb_id}/documents/batch-delete` | 批量删除文档 |
| POST | `/api/knowledge-base/test-retrieval` | 检索测试（请求体含 `kb_id`、`query` 等） |

## 文档处理流程

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 上传阶段                                                       │
│     用户上传文档 → MinIO 临时存储 → DocumentUpload 记录            │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. 后台处理 (process_document_background)                        │
│     asyncio.to_thread 避免阻塞事件循环                            │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. 文档解析                                                       │
│     按扩展名选择加载器:                                            │
│     - PyPDFLoader (.pdf)                                          │
│     - Docx2txtLoader (.docx)                                     │
│     - UnstructuredMarkdownLoader (.md)                          │
│     - TextLoader (.txt)                                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. 文本分块                                                       │
│     RecursiveCharacterTextSplitter                               │
│     - chunk_size: 1000（默认；`replace` 与预览可指定）            │
│     - chunk_overlap: 200（默认；须小于 chunk_size）               │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  5. 向量化                                                         │
│     EmbeddingsFactory.create() → Chroma 向量库                   │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  6. 完成                                                           │
│     文件移至永久存储 → Document + DocumentChunk 记录               │
└─────────────────────────────────────────────────────────────────┘
```

## 增量更新机制

### 两条路径

| 场景 | 入口 | 实现 |
|------|------|------|
| 新文件首次入库 | `POST …/documents/process` → `process_document_background` → `_process_document_background_sync` | **全量**：新建 `Document`、写入全部分块与向量 |
| 已入库文档换文件（同名覆盖） | `POST …/documents/{doc_id}/replace`（可选 Query 指定分块参数） | 覆盖 MinIO 后调用 **`process_document`**：按块 hash 对比 `ChunkRecord`（`app/shared/chunk_record.py`），只增删变更块并重算对应向量；分块大小与重叠可经 Query 传入，与 `process_document` 一致 |

分块 id 与全量路径一致：`sha256(f"{kb_id}:{file_name}:{块正文}")`（见 `document_processor.py`）。

### 文件去重（首次上传）

`upload_kb_documents` 中按同库内 `file_name + file_hash` 查已有 `Document`，已存在则返回 `skip_processing`，不重复建任务。
