## 模块概述

知识库模块负责文档的上传、解析、分块、向量化存储和检索测试。

**源码位置**: `app/modules/knowledge/`

## API 端点

### 知识库 CRUD

| 方法 | 端点 | 功能 |
|------|------|------|
| POST | `/api/knowledge-base/` | 创建知识库 |
| GET | `/api/knowledge-base/` | 知识库列表 |
| GET | `/api/knowledge-base/{kb_id}` | 知识库详情 |
| DELETE | `/api/knowledge-base/{kb_id}` | 删除知识库 |

## 文档处理流程

```
┌─────────────────────────────────────────────────────────────────┐
│  1. 上传阶段                                                       │
│     用户上传文档 → MinIO 临时存储 → DocumentUpload 记录            │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  2. 后台处理 (process_document_background)                        │
│     asyncio.to_thread 避免阻塞事件循环                            │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  3. 文档解析                                                       │
│     按扩展名选择加载器:                                            │
│     - PyPDFLoader (.pdf)                                          │
│     - Docx2txtLoader (.docx)                                     │
│     - UnstructuredMarkdownLoader (.md)                          │
│     - TextLoader (.txt)                                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  4. 文本分块                                                       │
│     RecursiveCharacterTextSplitter                               │
│     - chunk_size: 500                                            │
│     - chunk_overlap: 50                                          │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  5. 向量化                                                         │
│     EmbeddingsFactory.create() → Chroma 向量库                   │
└─────────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────────┐
│  6. 完成                                                           │
│     文件移至永久存储 → Document + DocumentChunk 记录               │
└─────────────────────────────────────────────────────────────────┘
```

## 增量更新机制

### ChunkRecord 类

通过 hash 对比实现增量同步：

```python
class ChunkRecord:
    """管理 DocumentChunk 的增量更新"""

    def add_chunks(self, chunks: list[DocumentChunk]):
        """仅插入新/变更块"""
        for chunk in chunks:
            if chunk.id not in self.existing_ids:
                self.db.add(chunk)

    def delete_chunks(self, chunk_ids: list[str]):
        """删除已移除的块"""
        for chunk_id in chunk_ids:
            if chunk_id in self.existing_ids:
                self.db.delete(...)  # 删除对应的 Chroma 向量
```

### 文件去重

上传时按 `file_name + file_hash` 检查，已存在则跳过。
