# 业务流程总览与索引

下文按用户旅程梳理主要流程，并指向实现位置。技术细节以代码与 [架构说明](../架构/后端项目架构说明.md) 为准。

## 1. 用户认证

1. **注册**：提交邮箱、用户名、密码 → 密码经 bcrypt 哈希后写入 `users`。
2. **登录**：校验用户名/密码 → 签发 JWT（[python-jose](https://python-jose.readthedocs.io/) 常用 HS256；生产环境务必更换 `SECRET_KEY`）。
3. **后续请求**：`Authorization: Bearer <token>`，依赖项解析 `user_id` 用于多租户隔离。

**路由前缀**：`/api/auth`（见 `app/modules/auth/routes.py`）。

## 2. 模型配置（LLM / 嵌入）

用户可保存多份 **LLM/嵌入配置**（`llm_embedding_configs`），并设置其中一份为当前启用（`users.active_llm_embedding_config_id`）。RAG 对话、文档向量化、评估任务在运行时会从数据库加载该配置。

详见 [RAG评估与配置说明.md](RAG评估与配置说明.md)。

**路由前缀**：`/api/llm-configs`。

## 3. 知识库与文档

1. **创建知识库**：名称、描述 → `knowledge_bases` 记录，按用户隔离。
2. **上传文档**：文件进入 **MinIO**，并产生 `document_uploads` / 处理任务元数据（依实现路径而定）。
3. **处理管线**：解析（PDF / DOCX / Markdown / 文本等）→ **分块**（`RecursiveCharacterTextSplitter` 等，见 [LangChain text splitters](https://python.langchain.com/docs/concepts/text_splitters/)）→ **嵌入** → 写入 **Chroma** collection，同时写入 `documents`、`document_chunks`。
4. **检索**：按对话关联的知识库构建检索器，查询向量库返回相关片段。

**路由前缀**：`/api/knowledge-base`。

## 4. 对话（RAG）

1. **创建对话**：绑定一个或多个知识库 ID。
2. **发送消息**：持久化用户消息与占位助手消息后，执行 RAG：
   - 历史感知：用 LLM **重写查询**（结合历史），再检索；
   - 将检索结果格式化为带 `[citation:N]` 的上下文；
   - **流式**输出答案（SSE，协议与前端约定一致）。
3. **结束后**：将完整助手回复写回消息记录。

**实现**：`app/modules/chat/rag_service.py`（LangChain LCEL，见 [LangChain 文档](https://python.langchain.com/docs/)）。

**路由前缀**：`/api/chat`。

## 5. RAG 评估（可选）

创建评估任务与测试用例，使用 **RAGAS**（[文档](https://docs.ragas.io/)）等指标对「问题—上下文—答案」打分；判分 LLM 与嵌入通过项目内工厂与运行时配置注入。

详见 [RAG评估与配置说明.md](RAG评估与配置说明.md)。

**路由前缀**：`/api/evaluation`。
