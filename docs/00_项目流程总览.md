# RAG Engine 流程总览

## 三大核心流程

本文档集包含以下流程说明：

| 序号 | 文档 | 描述 |
|------|------|------|
| 01 | [用户认证流程说明](./01_用户认证流程说明.md) | JWT + bcrypt 认证，Token 管理，API Key |
| 02 | [文档上传与向量化流程说明](./02_文档上传与向量化流程说明.md) | MinIO 存储 → 后台处理 → Chroma/Qdrant 向量库 |

## 流程串联关系

```
┌──────────────────────────────────────────────────────┐
│                  用户认证 (01)                        │
│                                                      │
│  登录成功 → JWT Token → 存储在 localStorage           │
│                     ↓                                │
│              后续所有请求携带 Bearer Token             │
└──────────────────────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────┐
│             文档上传与向量化 (02)                      │
│                                                      │
│  创建知识库 → 上传文档 → MinIO 临时存储               │
│       ↓                         ↓                    │
│  MySQL (KB)           后台处理 (Background)           │
│       ↓                         ↓                    │
│  文档元数据           解析 → 分块 → Embedding → 向量库 │
└──────────────────────────────────────────────────────┘
                          │
                          ▼
┌──────────────────────────────────────────────────────┐
│                RAG 对话 (03)                          │
│                                                      │
│  创建对话 (关联 KB) → 发送消息                        │
│       ↓                    ↓                         │
│  MySQL (chat)         SSE 流式响应                   │
│                       /         \                    │
│                      /           \                   │
│              相似度检索      LLM 生成                 │
│                     \           /                    │
│                      \         /                     │
│                    引用格式化 → 前端展示              │
└──────────────────────────────────────────────────────┘
```

## 关键数据模型关系

```
User (1) ──┬── (N) KnowledgeBase (1) ──┬── (N) Document (1) ──┬── (N) DocumentChunk
           │                           │                      │
           │                           │                      └──→ Chroma/Qdrant (向量)
           │                           │
           │                           └── (N) ProcessingTask → 状态: pending/processing/completed/failed
           │
           └── (N) Chat (1) ──┬── (N) Message
                              │
                              └── (N) KnowledgeBase (多对多 via chat_knowledge_bases)
```

## 统一入口

| 功能 | 入口 | 认证方式 |
|------|------|----------|
| 用户注册 | `POST /api/auth/register` | 无 |
| 用户登录 | `POST /api/auth/token` | 无 |
| 知识库 CRUD | `/api/knowledge-base/*` | JWT Bearer |
| 文档上传 | `POST /api/knowledge-base/{id}/documents/upload` | JWT Bearer |
| 文档处理 | `POST /api/knowledge-base/{id}/documents/process` | JWT Bearer |
| 对话 CRUD | `/api/chat/*` | JWT Bearer |
| 发送消息 | `POST /api/chat/{id}/messages` | JWT Bearer |
| 外部集成 | 各 API 端点 | `X-API-Key` Header |
