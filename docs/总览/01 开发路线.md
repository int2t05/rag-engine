# 开发路线（建议）

以下为从「能跑起来」到「可演示、可迭代」的推荐顺序，与当前代码结构一致。

## 第一阶段：环境与数据层

1. 使用 `docker-compose.infra.yml` 或自建服务启动 **MySQL、Chroma、MinIO**。
2. 配置项目根目录 `.env`（参考 `.env.example`），与 [Settings](https://docs.pydantic.dev/latest/concepts/pydantic_settings/) 字段对齐。
3. 启动后端，确认 `GET /api/health` 与 **Alembic 迁移** 在 `lifespan` 中执行成功（见 [FastAPI lifespan](https://fastapi.tiangolo.com/advanced/events/)）。

## 第二阶段：认证与模型配置

1. 完成注册/登录，确认 JWT 链路。
2. 在「模型配置」中保存并启用一套 **LLM + 嵌入**，确保 `active_llm_embedding_config_id` 有效。

## 第三阶段：知识库与 RAG

1. 创建知识库 → 上传文档 → 等待处理完成（向量写入 Chroma，元数据写入 MySQL）。
2. 创建对话并关联知识库 → 发消息验证 **流式回答** 与引用格式。

## 第四阶段：评估与运维（可选）

1. 接入 **RAGAS** 评估任务，核对指标与成本。
2. 生产环境：更换 `SECRET_KEY`、限制 CORS、使用 HTTPS、备份 MySQL 与 Chroma 卷等。
