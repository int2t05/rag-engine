# 用户认证流程说明

## 概述

本系统采用 **JWT (JSON Web Token)** 实现用户认证，密码使用 **bcrypt** 哈希存储，支持两种认证方式：
- **JWT Token**：适用于 Web 前端，通过 OAuth2 Password 模式获取
- **API Key**：适用于外部系统集成，通过 `X-API-Key` 请求头传递

## 核心代码

### 1. 注册 (`backend/app/api/api_v1/auth.py`)

```python
@router.post("/register", response_model=UserResponse)
def register(*, db: Session = Depends(get_db), user_in: UserCreate) -> Any:
    # 检查邮箱唯一性
    user = db.query(User).filter(User.email == user_in.email).first()
    if user:
        raise HTTPException(status_code=400, detail="使用此电子邮件的用户已存在。")

    # 检查用户名唯一性
    user = db.query(User).filter(User.username == user_in.username).first()
    if user:
        raise HTTPException(status_code=400, detail="具有此用户名的用户已存在.")

    # bcrypt 哈希密码后创建用户
    user = User(
        email=user_in.email,
        username=user_in.username,
        hashed_password=security.get_password_hash(user_in.password),  # bcrypt
    )
    db.add(user)
    db.commit()
    db.refresh(user)
    return user
```

### 2. 登录获取 Token (`backend/app/api/api_v1/auth.py`)

```python
@router.post("/token", response_model=Token)
def login_access_token(
    db: Session = Depends(get_db), form_data: OAuth2PasswordRequestForm = Depends()
) -> Any:
    # 按用户名查询用户
    user = db.query(User).filter(User.username == form_data.username).first()
    if not user or not security.verify_password(form_data.password, user.hashed_password):
        raise HTTPException(status_code=401, detail="用户名或密码不正确")

    # 生成 JWT Token，默认7天过期
    access_token_expires = timedelta(minutes=settings.ACCESS_TOKEN_EXPIRE_MINUTES)
    access_token = security.create_access_token(
        data={"sub": user.username}, expires_delta=access_token_expires
    )
    return {"access_token": access_token, "token_type": "bearer"}
```

### 3. 密码哈希 (`backend/app/core/security.py`)

```python
def verify_password(plain_password: str, hashed_password: str) -> bool:
    """bcrypt 自动处理 salt，无需手动管理"""
    return bcrypt.checkpw(
        plain_password.encode("utf-8"), hashed_password.encode("utf-8")
    )

def get_password_hash(password: str) -> str:
    """同一密码每次哈希结果不同，但验证时都能匹配"""
    return bcrypt.hashpw(password.encode("utf-8"), bcrypt.gensalt()).decode("utf-8")
```

### 4. Token 验证 (`backend/app/core/security.py`)

```python
def get_current_user(
    db: Session = Depends(get_db), token: str = Depends(oauth2_scheme)
) -> User:
    credentials_exception = HTTPException(
        status_code=401, detail="Could not validate credentials",
        headers={"WWW-Authenticate": "Bearer"},
    )
    try:
        payload = jwt.decode(token, settings.SECRET_KEY, algorithms=[settings.ALGORITHM])
        username: str = payload.get("sub")
        if username is None:
            raise credentials_exception
    except JWTError:
        raise credentials_exception

    user = db.query(User).filter(User.username == username).first()
    if user is None or not user.is_active:
        raise credentials_exception
    return user
```

### 5. API Key 认证 (`backend/app/core/security.py`)

```python
api_key_header = APIKeyHeader(name="X-API-Key", auto_error=False)

def get_api_key_user(db: Session = Depends(get_db), api_key: str = Security(api_key_header)) -> User:
    if not api_key:
        raise HTTPException(status_code=401, detail="API密钥头丢失")
    api_key_obj = APIKeyService.get_api_key_by_key(db=db, key=api_key)
    if not api_key_obj or not api_key_obj.is_active:
        raise HTTPException(status_code=401, detail="未激活API密钥")
    APIKeyService.update_last_used(db=db, api_key=api_key_obj)
    return api_key_obj.user
```

## 前端 Token 存储

```typescript
// frontend/src/lib/api.ts
export const authApi = {
  login: async (username: string, password: string) => {
    const response = await fetch("/api/auth/token", {
      method: "POST",
      headers: { "Content-Type": "application/x-www-form-urlencoded" },
      body: new URLSearchParams({ username, password }),
    });
    const data = await response.json();
    localStorage.setItem("token", data.access_token);  // ⚠️ 存储在 localStorage
    return data;
  },
};
```

## 安全改进建议

### 🔴 高优先级

| 问题 | 风险 | 建议 |
|------|------|------|
| **Token 存储在 localStorage** | 易受 XSS 攻击，攻击者可直接读取 Token | 改用 `httpOnly` Cookie 存储 Token，前端通过 `credentials: "include"` 发送 |
| **无 Refresh Token** | Access Token 过期（7天）后用户需重新登录 | 引入 Refresh Token 机制，双 Token 认证 |
| **无密码强度验证** | 后端对密码强度无校验，可设置弱密码 | 添加密码复杂度规则：至少8位，含大小写字母、数字、特殊字符 |

### 🟡 中优先级

| 问题 | 风险 | 建议 |
|------|------|------|
| `SECRET_KEY` 可能为默认值 `"change-me"` | 生产环境泄露风险 | 启动时强制校验 `SECRET_KEY` 非默认值 |
| 无登录失败限流 | 可被暴力破解 | 引入登录失败计数，5次失败后锁定15分钟 |
| 无注册邮箱验证 | 可批量注册无效账户 | 添加邮箱验证流程 |

### 🟢 低优先级

| 问题 | 建议 |
|------|------|
| 无登录日志审计 | 记录每次登录的 IP、UA、时间 |
| 无多设备登录管理 | 支持查看/吊销活跃会话 |
