# 05 · 模型配置、成本与评估

## 1. 为什么把 LLM / 嵌入配置放在数据库里，而不是只写环境变量？

**答题要点**

- **多用户/多环境**：不同团队用不同密钥或供应商；测试与生产隔离。
- **运维**：密钥轮换不必改镜像或重启发版（配合安全策略）。
- **产品**：未来可做「套餐」——不同订阅用不同模型档位（扩展点）。
- **业务话术**：「模型连接信息是**运营配置项**，不是每次改代码。」

**延伸：ContextVar 请求级注入实现（伪代码）**

```python
# app/shared/ai_runtime_context.py
# 每个请求/任务有独立的 AI 运行时配置，互不干扰
_ai_runtime_var: ContextVar[Optional[AiRuntimeSettings]] = ContextVar("_ai_runtime_var", default=None)

def get_ai_runtime() -> AiRuntimeSettings:
    v = _ai_runtime_var.get()
    if v is None:
        raise RuntimeError("未注入 AI 运行时配置")
    return v

# 请求入口处注入
async def chat_endpoint(request: ChatRequest, user: User):
    # 从数据库加载用户激活的配置，注入到 ContextVar
    config = load_user_active_config(user.id)  # 包含 openai_api_key / ollama_model 等
    token = set_ai_runtime_token(config)
    try:
        llm = LLMFactory.create(streaming=True)
        embeddings = EmbeddingsFactory.create()
        # ... 后续 RAG 流程
    finally:
        reset_ai_runtime_token(token)
```

- 用户激活 `active_llm_embedding_config_id`；接口 `/api/llm-configs`；详见 [../业务流程/RAG评估与配置说明.md](../业务流程/RAG评估与配置说明.md)。

## 2. 从业务视角，一次对话的主要成本构成是什么？

**答题要点**

| 成本项 | 说明 |
|--------|------|
| **嵌入** | 查询（及可能的改写子查询）要向量化 |
| **检索** | 向量库与可选 BM25/多路，带来计算与延迟 |
| **重排** | 交叉编码器额外计算 |
| **生成** | 按 Token 计费，上下文越长越贵 |
| **评估任务** | 批量调用，需单独预算与配额管理 |

**加分**：提出 **缓存**（同问题）、**限流**、**高峰降级**（关多路/重排）等产品策略。

## 3. RAGAS（或类似评估）能证明什么、不能证明什么？

**答题要点**

- **能**：在同一评估集上**对比**不同配置/版本；看趋势是否变好；适合**回归**。
- **不能**：
  - 不等于真实用户满意度；
  - 不等于业务 ROI；
  - 指标可能被「短答」「复制粘贴」等技巧影响，需**结合人工抽样**。

**延伸：RAGAS 五大指标与本项目实现**

| 指标 | 含义 | 本项目字段 |
|------|------|-----------|
| **faithfulness** | 生成答案是否忠于检索上下文 | `faithfulness` |
| **answer_relevancy** | 答案与问题的相关性 | `answer_relevancy` |
| **context_precision** | 检索片段的排序质量 | `context_precision` |
| **context_recall** | 检索覆盖参考答案的程度 | `context_recall` |
| **answer_correctness** | 答案相对参考答案的整体正确性 | `answer_correctness` |

## 4. 你如何定义「从 demo 到可交付业务」的上线标准？

**答题要点（可分层）**

- **基线**：核心场景**手工用例**通过（覆盖高频问题类型）。
- **质量**：评估集上关键指标不低于上一版本或不低于约定阈值。
- **性能**：P95 延迟、失败率可接受。
- **运营**：知识更新 SOP、失败工单路径、空检索话术。
- **安全**：密钥管理、备份（MySQL / Chroma / MinIO）有责任人。

---

## 5. 业务方要求「100% 准确」，你怎么回应？

**答题要点**

- **诚实**：RAG + LLM 无法承诺数学式 100%，尤其是开放域与强推理场景。
- **替代方案**：关键条款**人工审核发布**、回答模板化、仅检索展示不生成、或对高风险类问题**强制引用原文不总结**。
- **合同/对内**：用 **SLA** 描述可用性、响应时间、问题分级，而不是「100% 正确」。

---

## 6. 追问：「如何向财务解释向量库和对象存储的费用？」

**答题要点**

- **对象存储**：原始文件占用，随文档量线性增长。
- **向量**：随分块数与维度增长；重建索引会有临时计算成本。
- **对话**：按 Token 与调用次数，与用户数、多轮深度、是否开多路/重排强相关。

## 7. 延伸：工厂模式伪代码——如何实现 OpenAI / Ollama 无感切换

```python
# app/shared/llm/llm_factory.py
class LLMFactory:
    @staticmethod
    def create(provider: Optional[str] = None, temperature: float = 0, streaming: bool = True):
        s = get_ai_runtime()  # 从 ContextVar 获取当前请求配置
        provider = (provider or s.chat_provider).lower()

        if provider == "openai":
            # 支持 DeepSeek / 智谱等 OpenAI 兼容 API
            return init_chat_model(
                model=s.openai_model,
                model_provider="openai",
                api_key=s.openai_api_key,
                base_url=_openai_base(s.openai_api_base),
                temperature=temperature,
                streaming=streaming,
            )
        elif provider == "ollama":
            return ChatOllama(
                model=s.ollama_model,
                base_url=_ollama_host(s.ollama_api_base),
                temperature=temperature,
            )

# app/shared/embedding/embedding_factory.py
class EmbeddingsFactory:
    @staticmethod
    def create() -> Embeddings:
        s = get_ai_runtime()
        if s.embeddings_provider == "openai":
            return OpenAIEmbeddings(
                api_key=_openai_embeddings_api_key(),
                base_url=_openai_embeddings_base_url(),
                model=s.openai_embeddings_model,
                tiktoken_enabled=False,  # 减少网关兼容问题
            )
        return OllamaEmbeddings(
            model=s.ollama_embeddings_model,
            base_url=_ollama_embeddings_base_url(),
        )
```

## 小结（背诵用）

- **配置入库** = 灵活与运维友好，ContextVar 保证请求隔离。
- 成本 = **嵌入 + 检索 + 重排 + 生成 +（评估）**。
- RAGAS 五维度：faithfulness、answer_relevancy、context_precision、context_recall、answer_correctness。
- 评估 = **迭代工具**，上线标准要**多维**（质量、性能、运营、安全）。
