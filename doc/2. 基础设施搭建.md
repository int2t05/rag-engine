## 2.3 docker-compose.dev.yml（开发用）

#### 无需 Dockerfile（直接用官方 / 第三方镜像）

如果你的服务能直接使用 Docker Hub 等仓库的现成镜像（比如 MySQL、Redis、Nginx 等），就不需要写 Dockerfile，直接在 `docker-compose.yml` 中指定镜像名称即可。


```yaml
services:
  db:
    image: mysql:8.0
    command: --default-authentication-plugin=mysql_native_password
    environment:
      MYSQL_ROOT_PASSWORD: root
      MYSQL_DATABASE: ragengine
      MYSQL_USER: ragengine
      MYSQL_PASSWORD: ragengine
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
    networks:
      - app_network

  chromadb:
    image: chromadb/chroma:latest
    ports:
      - "8001:8000"
    volumes:
      - chroma_data:/chroma/chroma
    networks:
      - app_network

  minio:
    image: minio/minio:latest
    ports:
      - "9000:9000"
      - "9001:9001"
    environment:
      MINIO_ROOT_USER: minioadmin
      MINIO_ROOT_PASSWORD: minioadmin
    volumes:
      - minio_data:/data
    command: server --console-address ":9001" /data
    networks:
      - app_network

  nginx-dev:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./nginx.dev.conf:/etc/nginx/nginx.conf:ro
    depends_on:
      - backend
      - frontend
    networks:
      - app_network

volumes:
  mysql_data:
  chroma_data:
  minio_data:

networks:
  app_network:
    driver: bridge
```

*注：开发时 backend 和 frontend 可先不放进 compose，本地 `uvicorn` 和 `pnpm dev` 跑。若只用基础设施，可创建下面的 docker-compose.infra.yml。*

## 2.4 nginx.conf（简化版）

```nginx
events { worker_connections 1024; }

http {
    client_max_body_size 100M;
    upstream backend { server backend:8000; }
    upstream frontend { server frontend:3000; }

    server {
        listen 80 default_server;
        server_name _;

        location /api {
            proxy_pass http://backend/api;
            proxy_set_header Host $host;
            proxy_set_header X-Real-IP $remote_addr;
        }

        location / {
            proxy_pass http://frontend;
            proxy_set_header Host $host;
            proxy_http_version 1.1;
            proxy_set_header Upgrade $http_upgrade;
            proxy_set_header Connection "upgrade";
        }
    }
}
```

## 2.5 仅启动基础设施（不包含应用）

创建 `docker-compose.infra.yml`，只包含 db、chromadb、minio，便于本地开发时单独启动（此时 Nginx 不启动，因无 backend/frontend）：

```yaml
services:
  db:
    image: mysql:8.0
    # ... 同上
  chromadb:
    # ... 同上
  minio:
    # ... 同上
volumes:
  mysql_data:
  chroma_data:
  minio_data:
networks:
  app_network:
    driver: bridge
```

启动命令：

如果是本地 MySQL 服务占用，直接停止服务：

运行

```
sc stop mysql
```


临时暂停（快速恢复）

`docker compose -f docker-compose.infra.yml pause`

容器进入 “暂停” 状态（进程冻结），秒级恢复正常停止（保留容器）

`docker compose -f docker-compose.infra.yml stop`

容器停止运行（进程退出），但容器本身不删除彻底销毁（删容器）

`docker compose -f docker-compose.infra.yml down`停止 + 删除容器（但保留镜像 / 数据卷）

docker compose -f docker-compose.infra.yml down -v 全部删除

启动

```bash
docker compose -f docker-compose.infra.yml up -d
```
