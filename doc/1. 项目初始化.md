## 1.2 后端初始化

### 创建目录
__init__.py 标识Python包 `_init__.py` 在包被导入时会**自动执行**

- `__init__.py` 最核心的作用是**标记文件夹为 Python 包**，让文件夹内的模块可被导入；
- 其次是**控制包的对外接口**，简化用户导入逻辑，隐藏内部实现；
- 还能在包导入时**执行初始化逻辑**（如定义全局变量、加载配置），同时保证 Python 2/3 兼容性。


```bash
mkdir -p backend/app
touch backend/app/__init__.py
```

### backend/requirements.txt

`uvicorn` 是 Python 中基于 ASGI 标准的**高性能异步 Web 服务器**，主要用于运行 FastAPI、Starlette 等异步 Web 框架的应用。

`pydantic-settings` 是 Pydantic 官方的配置管理库，基于 Pydantic V2 构建，专门用于**读取和验证配置**（如环境变量、配置文件）。 自动从环境变量 /.env 文件加载配置并验证


```txt
fastapi>=0.104.1
uvicorn>=0.24.0
pydantic-settings>=2.0.0
```

### backend/app/main.py

```python
from fastapi import FastAPI

app = FastAPI(title="RAG Web UI", version="0.1.0")

@app.get("/")
def root():
    return {"message": "Welcome to RAG Web UI API"}

@app.get("/api/health")
def health():
    return {"status": "healthy"}
```

### 启动后端

```bash
cd backend
pip install -r requirements.txt
uvicorn app.main:app --reload --host 0.0.0.0 --port 8000
```

访问 http://localhost:8000 和 http://localhost:8000/docs 验证。

## 1.3 前端初始化

### 使用 create-next-app（推荐）

```bash
npx create-next-app@14 frontend --typescript --tailwind --eslint --app --src-dir
cd frontend
pnpm install
```

或手动创建 `package.json`：

```json
{
  "name": "rag-web-ui-frontend",
  "private": true,
  "scripts": {
    "dev": "next dev",
    "build": "next build",
    "start": "next start"
  },
  "dependencies": {
    "next": "14",
    "react": "^18",
    "react-dom": "^18"
  },
  "devDependencies": {
    "@types/node": "^20",
    "@types/react": "^18",
    "typescript": "^5",
    "tailwindcss": "^3"
  }
}
```

### src/app/page.tsx（最简首页）

```tsx
export default function Home() {
  return (
    <main className="min-h-screen p-8">
      <h1 className="text-2xl font-bold">RAG Web UI</h1>
      <p className="mt-4 text-gray-600">Welcome. This is the frontend.</p>
    </main>
  );
}
```

### 启动前端

- **`next dev`**：Next.js 框架自身的开发命令，启动开发服务器，支持热更新（HMR）和错误报告。
- **`npm run dev`**：通过 npm 运行 `package.json` 中定义的 `dev` 脚本（通常就是 `next dev`）。区别在于 npm 的安装机制和速度。
- **`pnpm dev`**：通过 pnpm 运行同一个脚本。核心差异是 pnpm 采用**内容寻址存储**，依赖包只存一份，通过硬链接 / 符号链接管理，**极省磁盘、安装更快**，且依赖隔离更严格。

```bash
cd frontend
pnpm dev
```

访问 http://localhost:3000 验证。

### .gitignore

Git 的 `.gitignore` 默认是**模式匹配（glob 模式）**，而非绝对路径。只要文件名 / 目录名匹配，就会在整个项目中生效，这也是它设计的灵活之处。

```gitignore
.env
__pycache__/
node_modules/
.next/
.venv/
venv/
```
