## 1. 认证架构概述

RAG Engine 采用**双轨认证体系**，支持两种认证方式：

| 认证方式 | 适用场景 | 有效期 | 认证头 |
|---------|---------|--------|--------|
| **JWT Token** | 前端用户登录 | 可配置（默认7天） | `Authorization: Bearer <token>` |
| **API Key** | 外部系统调用 | 长期有效，直到被禁用 | `X-API-Key: <key>` |

## 2. 技术选型与核心概念

### 2.1 密码哈希：bcrypt

bcrypt 是一种专为密码哈希设计的算法，具有以下特点：

- **盐值自动管理**：每次哈希都会生成随机盐值，无需手动处理
- **计算成本可调**：可通过 cost factor 控制计算时间，防止暴力破解
- **输出长度固定**：无论输入长度，输出均为 60 字符

```python
# 哈希密码
hashed = bcrypt.hashpw(password.encode("utf-8"), bcrypt.gensalt())

# 验证密码
bcrypt.checkpw(plain_password.encode("utf-8"), hashed.encode("utf-8"))
```

### 2.2 JWT（JSON Web Token）

JWT 是一种开放标准（RFC 7519），用于在各方之间安全地传输信息。其结构为：

```
xxxxx.yyyyy.zzzzz
│     │     │
│     │     └─ Signature（签名）
│     └─ Payload（负载）
└─ Header（头部）
```

本项目 JWT Payload 结构：

```json
{
  "sub": "username",    // subject：用户名
  "exp": 1699999999     // expiration time：过期时间戳
}
```

### 2.3 OAuth2 Password Bearer

OAuth2 是一种授权框架，本项目使用其中的**密码模式（Password Grant）**：

```
用户凭证 → Token → 后续请求携带 Token
```

FastAPI 自动处理：
- Token URL：`/api/auth/token`
- 请求格式：`username=xxx&password=xxx`（form-data）
- 响应格式：`{"access_token": "xxx", "token_type": "bearer"}`

### 4.2 API 接口

**接口**：`POST /api/auth/register`

**请求体**：

```json
{
  "username": "john_doe",
  "email": "john@example.com",
  "password": "secure_password_123"
}
```

**响应**：

```json
{
  "id": 1,
  "username": "john_doe",
  "email": "john@example.com",
  "is_active": true,
  "is_superuser": false,
  "created_at": "2026-03-19T10:00:00Z",
  "updated_at": "2026-03-19T10:00:00Z"
}
```

**错误情况**：

| 状态码 | 错误信息 | 原因 |
|-------|---------|------|
| 400 | 使用此电子邮件的用户已存在 | 邮箱已被注册 |
| 400 | 具有此用户名的用户已存在 | 用户名已被占用 |
| 503 | 网络错误或服务器无法访问 | 数据库连接失败 |

### 4.3 后端核心代码

```python
@router.post("/register", response_model=UserResponse)
def register(*, db: Session = Depends(get_db), user_in: UserCreate):
    # 检查邮箱唯一性
    user = db.query(User).filter(User.email == user_in.email).first()
    if user:
        raise HTTPException(status_code=400, detail="使用此电子邮件的用户已存在。")

    # 检查用户名唯一性
    user = db.query(User).filter(User.username == user_in.username).first()
    if user:
        raise HTTPException(status_code=400, detail="具有此用户名的用户已存在.")

    # 创建用户，密码自动哈希
    user = User(
        email=user_in.email,
        username=user_in.username,
        hashed_password=security.get_password_hash(user_in.password),  # bcrypt 哈希
    )
    db.add(user)
    db.commit()
    db.refresh(user)
    return user
```

### 5.2 OAuth2 密码模式

本项目使用 FastAPI 的 OAuth2PasswordRequestForm，符合 OAuth2 标准：

**接口**：`POST /api/auth/token`

**请求格式**：`application/x-www-form-urlencoded`

```
username=john_doe&password=secure_password_123
```

**响应**：

```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "token_type": "bearer"
}
```

### 5.3 后端核心代码

```python
@router.post("/token", response_model=Token)
def login_access_token(
    db: Session = Depends(get_db),
    form_data: OAuth2PasswordRequestForm = Depends()  # 自动解析 form-data
) -> Any:
    # 1. 查找用户
    user = db.query(User).filter(User.username == form_data.username).first()
    if not user:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="用户名或密码不正确",
        )

    # 2. 验证密码
    if not security.verify_password(form_data.password, user.hashed_password):
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="用户名或密码不正确",
        )

    # 3. 检查账户状态
    elif not user.is_active:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Inactive用户",
        )

    # 4. 生成 JWT Token
    access_token_expires = timedelta(minutes=settings.ACCESS_TOKEN_EXPIRE_MINUTES)
    access_token = security.create_access_token(
        data={"sub": user.username},  # Token 中存储用户名
        expires_delta=access_token_expires,
    )
    return {"access_token": access_token, "token_type": "bearer"}
```

### 6.2 依赖注入机制

FastAPI 的 `Depends()` 机制实现了认证的透明化，开发者只需在路由参数中声明即可：

```python
@router.get("/protected-endpoint")
def protected_route(
    current_user: User = Depends(security.get_current_user)  # 自动注入当前用户
):
    return {"user_id": current_user.id, "username": current_user.username}
```

### 6.3 后端核心代码

```python
def get_current_user(
    db: Session = Depends(get_db),
    token: str = Depends(oauth2_scheme)  # 自动从 Authorization 头提取
) -> User:
    credentials_exception = HTTPException(
        status_code=status.HTTP_401_UNAUTHORIZED,
        detail="Could not validate credentials",
        headers={"WWW-Authenticate": "Bearer"},
    )

    try:
        # 1. 解码 JWT
        payload = jwt.decode(
            token,
            settings.SECRET_KEY,
            algorithms=[settings.ALGORITHM]  # HS256
        )
        username: str = payload.get("sub")  # 提取用户名
        if username is None:
            raise credentials_exception
    except JWTError:
        raise credentials_exception

    # 2. 查找用户
    user = db.query(User).filter(User.username == username).first()
    if user is None:
        raise credentials_exception

    # 3. 检查账户状态
    if not user.is_active:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="Inactive user",
        )

    return user
```

### 6.4 Token 创建详解

```python
def create_access_token(data: dict, expires_delta: Optional[timedelta] = None) -> str:
    to_encode = data.copy()
    if expires_delta:
        expire = datetime.utcnow() + expires_delta
    else:
        expire = datetime.utcnow() + timedelta(
            minutes=settings.ACCESS_TOKEN_EXPIRE_MINUTES  # 默认 10080 分钟 = 7 天
        )

    to_encode.update({"exp": expire})  # 添加过期时间声明

    # 使用 HS256 算法签名
    encoded_jwt = jwt.encode(
        to_encode,
        settings.SECRET_KEY,  # 密钥（从 .env 配置）
        algorithm=settings.ALGORITHM
    )
    return encoded_jwt
```

```python
# API Key 请求头配置
api_key_header = APIKeyHeader(name="X-API-Key", auto_error=False)


def get_api_key_user(
    db: Session = Depends(get_db),
    api_key: str = Security(api_key_header),  # 从 X-API-Key 头提取
) -> User:
    if not api_key:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="API密钥头丢失",
        )

    # 1. 查找 API Key
    api_key_obj = APIKeyService.get_api_key_by_key(db=db, key=api_key)
    if not api_key_obj:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="未激活API密钥",
        )

    # 2. 检查激活状态
    if not api_key_obj.is_active:
        raise HTTPException(
            status_code=status.HTTP_401_UNAUTHORIZED,
            detail="未激活API密钥",
        )

    # 3. 更新最后使用时间（审计用）
    APIKeyService.update_last_used(db=db, api_key=api_key_obj)

    # 4. 返回关联的用户
    return api_key_obj.user
```

### 7.4 API Key 生成

API Key 由系统生成，格式建议为：

```
rag_sk_<随机字符串>
```

存储时只存储哈希值，不存储明文：

```python
import secrets

def generate_api_key() -> str:
    # 生成随机 API Key
    key = f"rag_sk_{secrets.token_urlsafe(32)}"
    return key

def hash_api_key(api_key: str) -> str:
    # 存储时哈希
    return bcrypt.hashpw(api_key.encode(), bcrypt.gensalt()).decode()
```

```typescript
// frontend/src/lib/api.ts
export async function fetchApi(url: string, options: RequestInit = {}) {
  const token = localStorage.getItem("token");

  const headers: Record<string, string> = {
    ...(token && { Authorization: `Bearer ${token}` }),  // 自动附加
    ...(options.headers as Record<string, string>),
  };

  const res = await fetch(`${API_BASE}${url}`, {
    ...options,
    headers,
  });

  // 401 处理：清除 Token 并跳转登录
  if (res.status === 401) {
    localStorage.removeItem("token");
    window.location.href = "/login";
    throw new ApiError(401, "登录已过期，请重新登录");
  }

  return res.json();
}
```

## 9. 安全最佳实践

### 9.1 密码安全

| 实践 | 说明 |
|-----|------|
| **使用 bcrypt** | 专为密码设计的哈希算法，自动盐值 |
| **最小长度** | 至少 6 位，建议 12 位以上 |
| **不存储明文** | 数据库只存哈希值，永远不存明文密码 |
| **错误信息模糊化** | 登录失败不区分"用户名不存在"和"密码错误" |

### 9.2 JWT 安全

| 实践 | 说明 |
|-----|------|
| **短期有效** | Token 设置合理过期时间（默认 7 天） |
| **密钥安全** | `SECRET_KEY` 使用强随机字符串，至少 32 字符 |
| **HTTPS 传输** | 生产环境必须使用 HTTPS |
| **敏感信息** | Token 只存 username，不存密码等敏感信息 |

### 9.3 API Key 安全

| 实践 | 说明 |
|-----|------|
| **只存哈希** | 数据库只存 API Key 的哈希值 |
| **记录使用** | 记录 `last_used_at` 用于审计 |
| **可禁用** | 支持禁用 Key，无需删除 |
| **隔离权限** | API Key 应关联特定用户，限制权限范围 |

### 9.4 配置建议

```env
# .env 示例
SECRET_KEY=your-super-secret-key-at-least-32-characters-long
ALGORITHM=HS256
ACCESS_TOKEN_EXPIRE_MINUTES=10080  # 7天，建议更短
```

## 10. 常见问题

### Q1: 登录失败返回 401，但不确定是用户名还是密码错误？

本项目为防止用户枚举攻击，登录失败时统一返回"用户名或密码不正确"。这是安全设计，不区分具体原因。

### Q2: Token 过期后如何处理？

前端收到 401 响应后，自动清除 localStorage 中的 Token 并跳转到登录页。用户需要重新登录。

### Q3: 如何修改 Token 过期时间？

修改 `.env` 文件中的 `ACCESS_TOKEN_EXPIRE_MINUTES`：

```env
ACCESS_TOKEN_EXPIRE_MINUTES=30  # 30 分钟
```

### Q4: API Key 和 JWT Token 可以同时使用吗？

可以，但针对不同场景：

| 场景 | 推荐方式 |
|-----|---------|
| 前端用户登录 | JWT Token |
| 外部系统调用 | API Key |
| 内部微服务通信 | API Key |

### Q5: 如何禁用某个用户？

将 `is_active` 设置为 `false`，该用户将无法登录和访问任何 API。

### Q6: 忘记密码如何处理？

当前版本未实现"忘记密码"功能。如需添加，可考虑：

1. 发送重置邮件（需要邮件服务）
2. 验证码链接包含 JWT（短期有效）
3. 重置后生成新密码哈希
